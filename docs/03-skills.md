# Skills — 마크다운으로 만드는 자동화 레시피

## Skill이란?

Skill은 Claude Code에서 **반복 작업을 자동화**하기 위한 기능이다. 핵심은 단순하다: **마크다운 파일 하나가 곧 Skill**이다.

코드를 작성하거나 복잡한 설정을 할 필요 없이, 마크다운으로 "이렇게 해라"라는 지시문을 작성하면 Claude가 그대로 따른다.

## 노코드 도구와의 비교

| 노코드 도구 (Zapier, Make 등) | Claude Code Skill |
|---|---|
| GUI에서 워크플로우를 드래그앤드롭으로 구성 | 마크다운 파일에 텍스트로 지시사항 작성 |
| 트리거 → 액션 → 결과 흐름을 시각적으로 설계 | Claude가 지시문을 읽고 알아서 순서대로 실행 |
| 각 액션마다 설정 UI가 별도로 존재 | 자연어로 원하는 것을 설명하면 됨 |
| 특정 앱 연동이 필요 (API 키 등) | Claude가 이미 가진 도구(파일 읽기, 쓰기, 터미널 등)를 활용 |

> 노코드 도구가 "레고 블록 조립"이라면, Skill은 "레시피 카드를 건네주는 것"에 가깝다.

## Skill의 실체: 프롬프트 주입

"Skill을 실행한다"는 표현이 사용되지만, 실제로는 **별도의 프로그램이 실행되는 것이 아니다**.

```
1. 사용자가 "/day1-test-skill" 입력
2. Claude Code가 .claude/skills/day1-test-skill/SKILL.md 파일을 찾음
3. 그 내용을 컨텍스트 윈도우에 주입 (= 시스템 프롬프트에 추가)
4. Claude가 그 지시사항을 읽고 따라서 응답
```

즉, "실행"이라기보다는 **"지시문을 읽게 하는 것"**이다. 별도의 에이전트가 생성되는 것이 아니라, **같은 Claude가 추가 지시를 받는 것**이다.

## Skill 파일 구조

Skill은 프로젝트 안의 `.claude/skills/` 디렉토리에 저장된다.

```
프로젝트/
└── .claude/
    └── skills/
        ├── sync-report/
        │   ├── SKILL.md           ← 메인 지시문
        │   ├── references/        ← 참조 데이터 (선택)
        │   │   └── template.md
        │   └── scripts/           ← 실행 스크립트 (선택)
        │       └── gather-data.sh
        ├── daily-standup/
        │   └── SKILL.md
        └── code-review/
            └── SKILL.md
```

### SKILL.md 작성 예시

```markdown
# Daily Standup 정리

## 역할
당신은 프로젝트 매니저입니다.

## 할 일
1. 오늘의 git 커밋 로그를 확인하세요
2. 변경된 파일을 분석하세요
3. 다음 포맷으로 정리하세요:

### 오늘 한 일
- [항목]

### 내일 할 일
- [항목]

### 이슈
- [항목]
```

## Skill과 CLAUDE.md의 차이

둘 다 "마크다운으로 Claude에게 지시하는 것"이지만 용도가 다르다.

| | **CLAUDE.md** | **Skill** |
|---|---|---|
| **로딩 시점** | 매 세션 자동 | `/명령어`로 호출할 때만 |
| **용도** | 프로젝트 전체에 적용되는 규칙 | 특정 작업의 레시피 |
| **길이** | 짧게 유지 (항상 읽히니까 토큰 절약) | 길어도 OK (필요할 때만 로드) |
| **비유** | 회사 사규 | 업무 매뉴얼 |
| **예시** | "한국어로 답해", "테스트 먼저 실행" | "/sync", "/daily-standup" |
| **개수** | 레벨별 최대 3개 | 무제한 |

## Skill이 유용한 실제 시나리오

### 1. 정기 보고서 생성
매주 같은 포맷의 보고서를 작성해야 할 때 — Skill에 템플릿과 데이터 수집 지시를 넣어두면 `/weekly-report` 한 번으로 완성.

### 2. 코드 리뷰 체크리스트
PR을 올릴 때마다 보안, 성능, 코드 스타일을 체크해야 할 때 — `/code-review`로 일관된 기준의 리뷰를 받을 수 있음.

### 3. 온보딩 가이드
새 팀원이 프로젝트에 합류할 때 — `/onboarding`으로 프로젝트 구조 설명, 환경 설정, 첫 번째 태스크까지 안내.

## 실전 사례: /yt-digest — YouTube 팟캐스트 분석기

위의 예시들은 단순한 시나리오였다. 실제로 복잡한 Skill은 **헬퍼 스크립트와 조합**하면 훨씬 강력해진다.

### 문제 상황

3시간짜리 기술 팟캐스트를 자주 본다. 하지만 30분 보다가 나머지를 "나중에" 북마크하고 끝. 영상 내용을 텍스트로 검색하고, 특정 발언을 찾고, 질문할 수 있으면 좋겠다.

### 설계 선택: 독립 도구 vs Skill

| | 독립 Python 스크립트 + Claude API | Claude Code Skill |
|---|---|---|
| **LLM 비용** | API 호출마다 과금 | 추가 비용 없음 (Claude Code 세션 내) |
| **후속 질문** | 매번 재실행 | 대화로 자연스럽게 |
| **리포트 형식** | 고정 템플릿 | 상황에 맞게 유연 |

핵심 인사이트: **Claude Code 자체가 LLM**이다. 트랜스크립트를 컨텍스트에 올리면 별도 API 없이 분석, 요약, Q&A가 전부 가능하다.

### 구조

```
~/.claude/commands/yt-digest.md         ← Skill 마크다운 (워크플로우 정의)
~/.claude/scripts/yt-digest/
├── extract.py                           ← Python 헬퍼 (자막 추출 + 캐싱)
└── transcripts/                         ← 로컬 캐시 (영상 ID별)
```

- **extract.py**: YouTube URL → 자막 추출 → 60초 단위 그루핑 → JSON 캐싱
- **yt-digest.md**: 인자 파싱 → extract.py 실행 → 결과 분석 (4가지 모드)

### 사용법

```
/yt-digest https://youtu.be/xxx              ← 개요 (주요 주제 + 핵심 인사이트)
/yt-digest https://youtu.be/xxx report        ← 전체 구조화 리포트
/yt-digest https://youtu.be/xxx 01:00:00-01:30:00  ← 구간 요약
/yt-digest https://youtu.be/xxx Jensen이 중국에 대해 뭐라 했어?  ← Q&A
```

### 배울 점

1. **Skill은 마크다운만으로 끝나지 않는다** — 외부 스크립트와 결합하면 복잡한 워크플로우도 가능
2. **Claude Code = LLM** — 별도 API를 호출할 필요 없이 Skill이 데이터를 컨텍스트에 올리면 Claude가 알아서 분석
3. **캐싱이 핵심** — 같은 영상을 다시 분석할 때 추출 과정을 건너뛰고 즉시 분석 시작

이 패턴(헬퍼 스크립트 + Skill 마크다운)은 미팅 녹음 분석, PDF 논문 요약, 데이터 수집 + 리포트 등 다양한 시나리오에 동일하게 적용 가능하다.

## 핵심 정리

- Skill = **마크다운 파일** (코드 아님, 비개발자도 작성 가능)
- 실행 = **지시문을 컨텍스트에 주입** (별도 프로그램 실행 아님)
- CLAUDE.md = 항상 적용되는 규칙 / Skill = 필요할 때 부르는 레시피
- 헬퍼 스크립트와 결합하면 **외부 데이터 추출 + AI 분석 워크플로우** 구축 가능
- 복잡한 멀티에이전트가 필요하면 Skill이 아닌 **Subagent**와 **Agent Teams**를 사용

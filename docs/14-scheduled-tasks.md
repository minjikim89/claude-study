# Scheduled Tasks — Claude에게 반복 업무를 맡기는 예약 시스템

## Scheduled Tasks란?

매일 아침 Slack 메시지를 요약해달라고, 매주 금요일 프로젝트 보고서를 정리해달라고 — 같은 요청을 반복하고 있다면, 이제 **한 번 설정해두고 자동으로 돌아가게** 할 수 있다.

Scheduled Tasks는 Claude Desktop의 **Cowork** 기능에서 사용하는 예약 실행 시스템이다. 작업 지시를 한 번 작성하면, 설정한 주기에 맞춰 Claude가 알아서 실행하고 결과를 남긴다.

> 요약: Scheduled Tasks = **Claude에게 걸어두는 크론잡(cron job)**

## 노코드 도구와의 비교

| 노코드 도구 | Scheduled Tasks 대응 |
|---|---|
| **Zapier의 Schedule Trigger** | 시간 기반 자동 실행 (매시간, 매일, 매주) |
| **Make의 Scenario 예약** | 주기적으로 시나리오 실행 |
| **Google Apps Script의 시간 기반 트리거** | 정해진 시간에 스크립트 실행 |
| **n8n의 Cron Node** | 크론 표현식 기반 반복 실행 |

핵심 차이:
- 노코드 도구는 **미리 만든 워크플로우**를 반복 실행
- Scheduled Tasks는 **자연어 지시**를 반복 실행 — 워크플로우 설계 없이 "이렇게 해줘"만 쓰면 됨

## 어디서 실행되나

```
┌─ Claude Desktop (Cowork) ─────────────────────┐
│                                                │
│  Scheduled Task: "매일 오전 9시"                │
│       │                                        │
│       ▼                                        │
│  Claude가 독립 Cowork 세션으로 실행             │
│       │                                        │
│       ├── MCP 서버 접근 가능 (Slack, Drive 등)  │
│       ├── Plugin/Skill 사용 가능               │
│       └── 결과를 세션에 저장                    │
│                                                │
│  ⚠️ 조건: 컴퓨터가 켜져있고 + 앱이 열려있어야  │
└────────────────────────────────────────────────┘
```

**중요 제약**: Scheduled Tasks는 **내 컴퓨터에서 실행**된다. 컴퓨터가 잠자기 모드이거나 Claude Desktop 앱이 닫혀 있으면 건너뛰고, 다시 켜지면 자동으로 실행한다.

이 점에서 Zapier/Make와 근본적으로 다르다:
- Zapier/Make: 클라우드에서 실행 → 내 컴퓨터 상태와 무관
- Scheduled Tasks: 로컬에서 실행 → 앱이 켜져 있어야 함

## 실전 활용 시나리오

### 1. 매일 아침 브리핑
```
매일 오전 9시
→ Slack 주요 채널 메시지 요약
→ 오늘 캘린더 일정 정리
→ 어제 처리 안 된 이슈 리마인드
```

### 2. 주간 보고서
```
매주 금요일 오후 5시
→ Google Drive에서 이번 주 문서 변경사항 수집
→ 스프레드시트 데이터를 보고서 포맷으로 정리
→ 결과를 지정 폴더에 저장
```

### 3. 경쟁사/업계 동향 추적
```
매일
→ 특정 키워드로 뉴스 검색
→ 주요 발견 사항을 요약
→ 트렌드 변화가 있으면 하이라이트
```

### 4. 파일 정리
```
매주 월요일
→ 다운로드 폴더의 파일을 유형별로 분류
→ 30일 이상 된 임시 파일 정리 목록 작성
```

## 만드는 방법 2가지

### 방법 1: `/schedule` 명령어 (대화형)

Cowork에서 `/schedule`을 입력하면 Claude가 대화로 안내한다.

```
사용자: /schedule
Claude: 어떤 작업을 예약할까요?
사용자: 매일 아침 Slack 요약해줘
Claude: 실행 빈도를 선택하세요 → [매시간] [매일] [매주] [평일만]
...
→ "Schedule" 버튼 클릭으로 확정
```

### 방법 2: Scheduled Tasks 페이지 (직접 입력)

왼쪽 사이드바 **"Scheduled"** 클릭 → **"+ New task"** 클릭

| 필드 | 설명 |
|---|---|
| Task name | 작업 이름 (예: "Daily Slack Briefing") |
| Description | 짧은 설명 |
| Prompt | 상세 지시문 (Skill처럼 자연어로 작성) |
| Frequency | 매시간 / 매일 / 매주 / 평일만 / 수동 |
| Model | 사용할 모델 선택 (선택사항) |
| Working folder | 작업 디렉토리 (선택사항) |

## 관리 기능

Scheduled Tasks 페이지에서 할 수 있는 것:

- 실행 이력 확인 (성공/건너뜀/실패)
- 지시문이나 주기 수정
- 일시 정지 / 재개
- 수동으로 즉시 실행
- 삭제

## 사용 조건

| 항목 | 조건 |
|---|---|
| 플랜 | Pro, Max, Team, Enterprise |
| 플랫폼 | Claude Desktop만 (웹 불가) |
| 실행 조건 | 컴퓨터 깨어있음 + Claude Desktop 실행 중 |

## 핵심 정리

- Scheduled Tasks = **자연어 지시를 예약 실행**하는 크론잡
- Zapier/Make처럼 **워크플로우를 설계할 필요 없이** 말로 지시하면 됨
- 단, **로컬 실행**이라 컴퓨터가 켜져 있어야 한다는 제약이 있음
- MCP 서버, Plugin, Skill과 함께 사용 가능 → 외부 서비스 연동 자동화에 활용
- 클라우드 기반 24/7 자동화가 필요하면 여전히 Zapier/Make가 적합

# Remote Control — 내 컴퓨터의 Claude Code를 폰에서 이어하기

## Remote Control이란?

데스크에서 Claude Code로 작업하다가 자리를 비워야 할 때 — 소파에서 폰으로, 다른 컴퓨터 브라우저에서 **그 세션을 그대로 이어서** 쓸 수 있는 기능이다.

핵심은 **내 컴퓨터에서 계속 실행**된다는 점. 코드도, 파일도, MCP 서버도 전부 로컬 환경 그대로 유지되고, 폰이나 브라우저는 **그 화면을 보는 창 역할**만 한다.

> 요약: Remote Control = **내 터미널의 리모컨**

## 노코드 도구와의 비교

| 노코드 도구 | Remote Control 대응 |
|---|---|
| **Zapier 모바일 앱** | 실행 결과를 폰에서 확인 |
| **Notion 모바일 앱** | 데스크에서 쓰던 문서를 폰에서 이어 편집 |
| **Google Docs 모바일** | 어디서든 같은 문서에 접근 |
| **TeamViewer/원격 데스크톱** | 내 컴퓨터를 다른 기기에서 조작 |

핵심 차이:
- 노코드 앱은 **클라우드에 있는 데이터**에 접근 → 어디서든 동일
- Remote Control은 **내 컴퓨터에 실행 중인 세션**에 접근 → 로컬 파일, MCP 서버 그대로 사용

## 동작 원리

```
┌─ 내 컴퓨터 ───────────────────────────────────┐
│                                                │
│  Claude Code 세션 (로컬 실행)                   │
│  ├── 내 파일 시스템                             │
│  ├── MCP 서버 (Slack, DB 등)                   │
│  ├── Plugin/Skill                              │
│  └── 프로젝트 설정                              │
│       │                                        │
│       │ (HTTPS, 아웃바운드만)                    │
│       ▼                                        │
│  Anthropic API ◄─── 메시지 중계                 │
│                                                │
└────────────────────────────────────────────────┘
         ▲               ▲               ▲
         │               │               │
    ┌────┴────┐    ┌─────┴─────┐   ┌─────┴─────┐
    │ 터미널   │    │ 브라우저   │   │ 폰 앱     │
    │ (데스크) │    │ (다른 PC) │   │ (소파)    │
    └─────────┘    └───────────┘   └───────────┘
```

**보안 포인트**:
- 내 컴퓨터에서 **인바운드 포트를 열지 않음** (외부에서 직접 접속 불가)
- 모든 통신은 **HTTPS 아웃바운드**로 Anthropic API를 경유
- 일반 Claude Code 세션과 동일한 보안 수준

## Remote Control vs Claude Code on the Web

같은 claude.ai/code 화면을 쓰지만 **실행 위치가 다르다**.

| | Remote Control | Claude Code on the Web |
|---|---|---|
| **실행 위치** | 내 컴퓨터 | Anthropic 클라우드 |
| **로컬 파일 접근** | 가능 | 불가 |
| **MCP 서버** | 내 설정 그대로 사용 | 불가 |
| **적합한 상황** | 진행 중인 로컬 작업을 이어할 때 | repo 클론 없이 바로 시작할 때 |
| **병렬 작업** | 세션당 1개 | 여러 태스크 병렬 가능 |

> **쉬운 판단 기준**: "이미 작업 중" → Remote Control / "새로 시작" → Web

## 사용 방법

### 1. 새 세션으로 시작

```bash
claude remote-control
```

터미널에 **세션 URL**과 **QR 코드**가 표시된다. 스페이스바로 QR 코드 토글.

### 2. 진행 중인 세션에서 전환

이미 Claude Code를 사용 중일 때:

```
/remote-control
```

또는 줄여서:

```
/rc
```

현재 대화 이력을 그대로 유지한 채 원격 접속이 활성화된다.

### 3. 다른 기기에서 연결

세 가지 방법 중 선택:

| 방법 | 설명 |
|---|---|
| **URL 직접 입력** | 터미널에 표시된 세션 URL을 브라우저에서 열기 |
| **QR 코드 스캔** | 폰 카메라로 스캔 → Claude 앱에서 바로 연결 |
| **앱에서 찾기** | claude.ai/code 또는 모바일 앱에서 세션 목록 확인 (초록 점 = 온라인) |

### 4. 항상 Remote Control 켜두기

매번 명령어를 치지 않고, 모든 세션에서 자동 활성화하려면:

```
/config
→ "Enable Remote Control for all sessions" → true
```

## 제약 사항

| 제약 | 설명 |
|---|---|
| 세션 수 | 한 번에 1개 원격 세션만 가능 |
| 터미널 | 터미널을 닫으면 세션 종료 |
| 네트워크 | 약 10분 이상 네트워크 끊기면 세션 타임아웃 |
| 자동 복구 | 노트북 잠자기 → 깨어나면 자동 재연결 |

## 실전 활용 시나리오

### 1. 데스크 → 소파 이동
```
데스크에서 Claude Code로 버그 수정 중
→ /rc 입력 → QR 코드 스캔
→ 소파에서 폰으로 "이 파일도 같이 수정해줘" 지시
→ 내 컴퓨터에서 실행, 결과는 폰에서 확인
```

### 2. 장시간 작업 모니터링
```
큰 리팩토링 작업 시작
→ claude remote-control로 실행
→ 외출하면서 폰으로 진행 상황 확인
→ 필요하면 폰에서 추가 지시
```

### 3. 회의 중 빠른 확인
```
회의 중 코드 관련 질문을 받음
→ 폰에서 진행 중인 세션에 접속
→ "이 함수 구조 설명해줘" 요청
→ 로컬 코드를 직접 읽고 답변
```

| 항목 | 조건 |
|---|---|
| 플랜 | Max (Pro는 곧 지원 예정) |
| 인증 | claude.ai로 `/login` 완료 |
| 클라이언트 | 터미널의 Claude Code CLI |
| 연결 기기 | claude.ai/code 또는 iOS/Android Claude 앱 |

## 핵심 정리

- Remote Control = 로컬 Claude Code 세션의 **리모컨** (폰, 브라우저에서 접속)
- 실행은 **항상 내 컴퓨터**에서 → 로컬 파일, MCP 서버, Plugin 모두 사용 가능
- `claude remote-control` 또는 `/rc`로 활성화 → URL이나 QR 코드로 연결
- Claude Code on the Web과 혼동 주의: Remote Control은 **로컬 실행**, Web은 **클라우드 실행**
- 터미널이 열려 있어야 하고, 네트워크가 약 10분 이상 끊기면 세션 종료

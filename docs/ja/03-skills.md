# スキル — マークダウンで作る自動化レシピ

## スキルとは？

スキルは Claude Code で**繰り返し作業を自動化**するための機能です。核心はシンプルです：**マークダウンファイル 1 つがそのままスキル**になります。

コードを書いたり複雑な設定をしたりする必要はなく、マークダウンで「このようにしろ」という指示文を書けば、Claude がそのとおりに従います。

## ノーコードツールとの比較

| ノーコードツール（Zapier、Make など） | Claude Code スキル |
|---|---|
| GUI でワークフローをドラッグ＆ドロップで構成 | マークダウンファイルにテキストで指示内容を記述 |
| トリガー → アクション → 結果の流れを視覚的に設計 | Claude が指示文を読んで自分で順番に実行 |
| 各アクションごとに設定 UI が別途存在 | 自然言語でやりたいことを説明すればよい |
| 特定アプリの連携が必要（API キーなど） | Claude がすでに持つツール（ファイル読み書き、ターミナルなど）を活用 |

> ノーコードツールが「レゴブロックの組み立て」なら、スキルは「レシピカードを手渡すこと」に近いです。

## スキルの実体：プロンプト注入

「スキルを実行する」という表現が使われますが、実際には**別のプログラムが実行されるわけではありません**。

```
1. ユーザーが "/day1-test-skill" を入力
2. Claude Code が .claude/skills/day1-test-skill/SKILL.md ファイルを探す
3. その内容をコンテキストウィンドウに注入（= システムプロンプトに追加）
4. Claude がその指示内容を読んで従って応答
```

つまり「実行」というより**「指示文を読ませること」**です。別のエージェントが生成されるのではなく、**同じ Claude が追加の指示を受け取る**だけです。

## スキルのファイル構造

スキルはプロジェクト内の `.claude/skills/` ディレクトリに保存されます。

```
プロジェクト/
└── .claude/
    └── skills/
        ├── sync-report/
        │   ├── SKILL.md           ← メインの指示文
        │   ├── references/        ← 参照データ（任意）
        │   │   └── template.md
        │   └── scripts/           ← 実行スクリプト（任意）
        │       └── gather-data.sh
        ├── daily-standup/
        │   └── SKILL.md
        └── code-review/
            └── SKILL.md
```

### SKILL.md の作成例

```markdown
# Daily Standup まとめ

## 役割
あなたはプロジェクトマネージャーです。

## やること
1. 今日の git コミットログを確認してください
2. 変更されたファイルを分析してください
3. 以下のフォーマットでまとめてください：

### 今日やったこと
- [項目]

### 明日やること
- [項目]

### 課題
- [項目]
```

## スキルと CLAUDE.md の違い

どちらも「マークダウンで Claude に指示する」ものですが、用途が異なります。

| | **CLAUDE.md** | **スキル** |
|---|---|---|
| **ロードのタイミング** | 毎セッション自動 | `/コマンド` で呼び出したときのみ |
| **用途** | プロジェクト全体に適用されるルール | 特定作業のレシピ |
| **長さ** | 短く保つ（常に読まれるのでトークン節約） | 長くても OK（必要なときだけロード） |
| **例え** | 会社の就業規則 | 業務マニュアル |
| **使用例** | 「日本語で答えて」「テストを先に実行」 | `/sync`、`/daily-standup` |
| **数** | レベルごとに最大 3 つ | 無制限 |

## スキルが役立つ実際のシナリオ

### 1. 定期レポート生成
毎週同じフォーマットのレポートを作成しなければならないとき — スキルにテンプレートとデータ収集の指示を入れておけば、`/weekly-report` 一発で完成。

### 2. コードレビューチェックリスト
PR を上げるたびにセキュリティ、パフォーマンス、コードスタイルを確認しなければならないとき — `/code-review` で一貫した基準のレビューを受けられる。

### 3. オンボーディングガイド
新しいチームメンバーがプロジェクトに参加するとき — `/onboarding` でプロジェクト構造の説明、環境設定、最初のタスクまで案内。

## 実践事例：/yt-digest — YouTubeポッドキャスト分析ツール

上記のシナリオはシンプルなものでした。実際には、スキルは**ヘルパースクリプトと組み合わせる**とはるかに強力になります。

### 問題

3時間のテックポッドキャストをよく見る。でも30分見て残りを「後で」ブックマークして終わり。動画の内容を検索可能なテキストとして参照したり質問したりできればいいのに。

### 設計の選択：スタンドアロンツール vs スキル

| | スタンドアロンPython + Claude API | Claude Code スキル |
|---|---|---|
| **LLMコスト** | API呼び出しごとに課金 | 追加コストなし（Claude Codeセッション内） |
| **フォローアップ質問** | 毎回再実行 | 自然な会話の流れ |
| **レポート形式** | 固定テンプレート | 状況に応じて柔軟 |

核心的な気づき：**Claude Code自体がLLM**。トランスクリプトをコンテキストにロードすれば、分析・要約・Q&Aが別途API不要で全て可能。

### アーキテクチャ

```
~/.claude/commands/yt-digest.md         ← スキルマークダウン（ワークフロー定義）
~/.claude/scripts/yt-digest/
├── extract.py                           ← Pythonヘルパー（抽出＋キャッシュ）
└── transcripts/                         ← ローカルキャッシュ（動画ID別）
```

- **extract.py**：YouTube URL → 字幕抽出 → 60秒単位グルーピング → JSONキャッシュ
- **yt-digest.md**：引数解析 → extract.py実行 → 結果分析（4モード）

### 使い方

```
/yt-digest https://youtu.be/xxx              ← 概要（主要トピック＋インサイト）
/yt-digest https://youtu.be/xxx report        ← 詳細構造化レポート
/yt-digest https://youtu.be/xxx 01:00:00-01:30:00  ← 区間要約
/yt-digest https://youtu.be/xxx Jensenが中国について何と言った？  ← Q&A
```

### 学びのポイント

1. **スキルはマークダウンだけに限らない** — 外部スクリプトと組み合わせれば複雑なワークフローも可能
2. **Claude Code = LLM** — 別途APIを呼ぶ必要なし。スキルがデータをコンテキストにロードし、Claudeが分析
3. **キャッシュが鍵** — 同じ動画の再分析時は抽出をスキップして即座にインサイトへ

このパターン（ヘルパースクリプト＋スキルマークダウン）は、ミーティング録音分析、PDF論文要約、データ収集＋レポートなど様々なシナリオに同様に適用可能。

## まとめ

- スキル = **マークダウンファイル**（コードではない、非エンジニアでも作成可能）
- 実行 = **指示文をコンテキストに注入**（別のプログラムを実行するのではない）
- CLAUDE.md = 常に適用されるルール / スキル = 必要なときに呼ぶレシピ
- ヘルパースクリプトとの組み合わせで**外部データ抽出＋AI分析ワークフロー**の構築が可能
- 複雑なマルチエージェントが必要な場合は、スキルではなく **Subagent** と **Agent Teams** を使用する

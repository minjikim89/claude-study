# リモートコントロール — 自分のPCのClaude Codeをスマホで続ける

## リモートコントロールとは？

デスクでClaude Codeで作業していて、席を離れなければならないとき — ソファでスマホで、または別のコンピューターのブラウザで**そのセッションをそのまま続けて**使える機能です。

核心は**自分のコンピューターで実行し続ける**という点です。コードも、ファイルも、MCPサーバーも、すべてローカル環境のまま維持され、スマホやブラウザは**その画面を見るウィンドウの役割**のみを担います。

> まとめ: リモートコントロール = **自分のターミナルのリモコン**

## ノーコードツールとの比較

| ノーコードツール | リモートコントロールの対応 |
|---|---|
| **Zapierモバイルアプリ** | 実行結果をスマホで確認 |
| **Notionモバイルアプリ** | デスクで使っていたドキュメントをスマホで引き続き編集 |
| **Google Docsモバイル** | どこからでも同じドキュメントにアクセス |
| **TeamViewer/リモートデスクトップ** | 自分のコンピューターを別のデバイスから操作 |

主な違い:
- ノーコードアプリは**クラウドにあるデータ**にアクセス → どこからでも同じ
- リモートコントロールは**自分のコンピューターで実行中のセッション**にアクセス → ローカルファイル、MCPサーバーをそのまま使用

## 仕組み

```
┌─ 自分のコンピューター ─────────────────────────────┐
│                                                │
│  Claude Codeセッション（ローカル実行）              │
│  ├── 自分のファイルシステム                        │
│  ├── MCPサーバー（Slack、DBなど）                  │
│  ├── プラグイン/スキル                             │
│  └── プロジェクト設定                              │
│       │                                        │
│       │（HTTPS、アウトバウンドのみ）                │
│       ▼                                        │
│  Anthropic API ◄─── メッセージ中継                 │
│                                                │
└────────────────────────────────────────────────┘
         ▲               ▲               ▲
         │               │               │
    ┌────┴────┐    ┌─────┴─────┐   ┌─────┴─────┐
    │ ターミナル│    │ ブラウザ   │   │ スマホアプリ│
    │（デスク） │    │（別のPC）  │   │（ソファ）  │
    └─────────┘    └───────────┘   └───────────┘
```

**セキュリティポイント**:
- 自分のコンピューターで**インバウンドポートを開かない**（外部からの直接接続不可）
- すべての通信は**HTTPS アウトバウンド**でAnthropicのAPIを経由
- 通常のClaude Codeセッションと同じセキュリティレベル

## リモートコントロール vs Claude Code on the Web

同じ claude.ai/code の画面を使いますが、**実行場所が異なります**。

| | リモートコントロール | Claude Code on the Web |
|---|---|---|
| **実行場所** | 自分のコンピューター | Anthropicのクラウド |
| **ローカルファイルアクセス** | 可能 | 不可 |
| **MCPサーバー** | 自分の設定のままで使用 | 不可 |
| **適した状況** | 進行中のローカル作業を続けるとき | リポジトリのクローンなしですぐ始めるとき |
| **並列作業** | セッションあたり1つ | 複数タスクを並列実行可能 |

> **簡単な判断基準**: 「すでに作業中」→ リモートコントロール / 「新しく始める」→ Web

## 使い方

### 1. 新しいセッションとして開始

```bash
claude remote-control
```

ターミナルに**セッションURL**と**QRコード**が表示されます。スペースキーでQRコードをトグルできます。

### 2. 進行中のセッションから切り替え

すでにClaude Codeを使用中の場合:

```
/remote-control
```

または短縮形で:

```
/rc
```

現在の会話履歴をそのまま維持した状態でリモートアクセスが有効になります。

### 3. 別のデバイスから接続

3種類の方法から選択:

| 方法 | 説明 |
|---|---|
| **URLを直接入力** | ターミナルに表示されたセッションURLをブラウザで開く |
| **QRコードをスキャン** | スマホカメラでスキャン → Claudeアプリで直接接続 |
| **アプリで探す** | claude.ai/code またはモバイルアプリでセッション一覧を確認（緑の点 = オンライン） |

### 4. 常時リモートコントロールを有効にする

毎回コマンドを入力せずに、すべてのセッションで自動的に有効にするには:

```
/config
→ "Enable Remote Control for all sessions" → true
```

## 制限事項

| 制限 | 説明 |
|---|---|
| セッション数 | 一度に1つのリモートセッションのみ |
| ターミナル | ターミナルを閉じるとセッションが終了 |
| ネットワーク | 約10分以上ネットワークが切断するとセッションがタイムアウト |
| 自動復旧 | ノートPCのスリープ → 起動すると自動再接続 |

## 実践シナリオ

### 1. デスク → ソファへの移動
```
デスクでClaude Codeでバグ修正中
→ /rc を入力 → QRコードをスキャン
→ ソファでスマホから "このファイルも一緒に修正して" と指示
→ 自分のコンピューターで実行、結果はスマホで確認
```

### 2. 長時間作業のモニタリング
```
大規模なリファクタリング作業を開始
→ claude remote-control で実行
→ 外出しながらスマホで進捗を確認
→ 必要に応じてスマホから追加指示
```

### 3. 会議中のクイック確認
```
会議中にコードに関する質問を受けた
→ スマホで進行中のセッションにアクセス
→ "この関数の構造を説明して" とリクエスト
→ ローカルコードを直接読んで回答
```

## 使用条件

| 項目 | 条件 |
|---|---|
| プラン | Max（Proは近日サポート予定） |
| 認証 | claude.ai で `/login` 完了 |
| クライアント | ターミナルのClaude Code CLI |
| 接続デバイス | claude.ai/code またはiOS/Android Claudeアプリ |

## まとめ

- リモートコントロール = ローカルのClaude Codeセッションの**リモコン**（スマホ、ブラウザから接続）
- 実行は**常に自分のコンピューター**で → ローカルファイル、MCPサーバー、プラグインすべて使用可能
- `claude remote-control` または `/rc` で有効化 → URLかQRコードで接続
- Claude Code on the Webと混同しないこと: リモートコントロールは**ローカル実行**、Webは**クラウド実行**
- ターミナルが開いている必要があり、ネットワークが約10分以上切断するとセッションが終了

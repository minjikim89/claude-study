# 記憶体系 — Claude はどのように記憶するか

## 基本前提：Claude は会話が終わるとすべてを忘れる

ChatGPT や Claude の Web 版を使ったことがある方はご存知のとおり、AI は基本的に**セッションが終わると以前の会話を覚えていません**。毎回初対面のように会話を始める必要があります。

Claude Code はこの根本的な限界を克服するため、**ファイルベースの記憶体系**を構築しています。人で言えば「記憶力がない代わりに、こまめにメモを取る秘書」のようなものです。

## 記憶の寿命による分類

### 揮発性記憶（セッションが終わると消える）

| 種類 | 説明 | 例え |
|---|---|---|
| **会話内容（Messages）** | 今この瞬間やり取りしているメッセージ | 通話中の電話 |
| **ロードされたスキル内容** | `/コマンド` で呼び出した指示内容 | 今読んでいるマニュアル |

セッションを閉じるとこれらの内容は完全に消えます。次のセッションで「さっき言ったこと覚えてる？」と聞いても分かりません。

### 永続記憶（ファイルに保存、セッションをまたいで維持）

| 種類 | 説明 | 例え |
|---|---|---|
| **CLAUDE.md** | ユーザーが直接作成したルール | 会社の就業規則 |
| **Auto Memory** | Claude が自動的に書く学習ノート | 秘書の個人メモ帳 |
| **スキルファイル** | ディスクに保存された作業レシピ | レシピブック |

これらはマークダウンファイルとしてディスクに保存されるため、セッションが変わっても残り続けます。

## 永続記憶 3 種の比較

| | **CLAUDE.md** | **Auto Memory** | **スキル** |
|---|---|---|---|
| **何か** | ルールブック | 学習ノート | 業務レシピ |
| **例え** | 会社の就業規則 | 秘書の個人メモ帳 | 料理レシピブック |
| **誰が作成** | 人（手動） | Claude（自動）＋人 | 人（手動） |
| **ロードのタイミング** | 毎セッション自動 | 毎セッション自動 | 呼び出し時のみ |
| **保存場所** | プロジェクトルート / `~/.claude/` | `~/.claude/projects/<プロジェクト>/memory/` | `.claude/skills/` |
| **用途** | 「このようにしろ」 | 「これを学んだ」 | 「この作業はこのように」 |
| **使用例** | 「日本語で答えて」 | 「このユーザーはテーブルを好む」 | 「/sync でレポート生成」 |
| **編集権限** | ユーザーのみ | Claude 自動 ＋ ユーザー手動 | ユーザーのみ |

## Auto Memory の詳細

### 仕組み

Auto Memory は Claude が**会話中に発見したパターンを自動的にメモ**する機能です。

```
会話中に Claude がパターンを発見
  │
  ▼
「このユーザーはテーブル形式を好むな」
  │
  ▼
~/.claude/projects/<プロジェクト>/memory/MEMORY.md に自動記録
  │
  ▼
次のセッションで自動的に読み込まれる → 最初からテーブルにまとめて回答
```

### Auto Memory が記録するもの

- **ユーザーの好み**：レスポンス形式、言語、コーディングスタイル
- **プロジェクトのパターン**：よく使うファイルパス、アーキテクチャの決定事項
- **デバッグの洞察**：繰り返し発生した問題の解決方法
- **ワークフロー**：ユーザーが好む作業の順序

### Auto Memory が記録しないもの

- セッション限定のコンテキスト（進行中の作業、一時的な状態）
- 検証されていない推測
- CLAUDE.md と重複する内容

### 確認・編集

```bash
# /memory コマンドで Auto Memory を確認・編集
# Claude Code 内で実行

/memory
```

直接ファイルを開いて編集することもできます：
```
~/.claude/projects/<プロジェクト名>/memory/MEMORY.md
```

### ノーコードツールとの比較

| ノーコードツール | Claude Code Auto Memory |
|---|---|
| Notion AI の「自分の文章スタイルを学習」 | ユーザーの好みパターンを学習 |
| Zapier の履歴ベースの推薦 | 前のセッションの学習を次のセッションに反映 |
| ChatGPT の「Memory」機能 | ほぼ同じコンセプト、ただしファイルベースで透明に管理 |

ChatGPT Memory との大きな違い：Claude Code の Auto Memory は**マークダウンファイル**なので、内容を直接確認・編集できます。ブラックボックスではありません。

## 一言まとめ

```
CLAUDE.md  = 私が Claude に渡すルール         （常にロード）
Memory     = Claude が自ら書く学習ノート       （常にロード）
スキル     = 特定作業のレシピ                 （呼び出し時にロード）
会話       = 今この瞬間のコンテキスト           （セッション終了で消滅）
```

> この 4 つの階層は結局**コンテキストウィンドウ（200k トークン）を効率的に使うための設計**です。
> 常に必要なものは自動ロード、たまに必要なものはオンデマンドロード、一時的なものは会話にのみ存在。

## まとめ

- Claude は基本的に記憶を持たない → ファイルで記憶を補完する構造
- 揮発性（会話）vs 永続（CLAUDE.md、Auto Memory、スキル）の 2 層で区別
- Auto Memory は Claude が自動的に学習した内容をファイルに記録 → 次のセッションに反映
- すべての記憶はマークダウンファイルなので透明に確認・編集可能
- コンテキストウィンドウ 200k トークンの効率的な使用が記憶体系設計の核心目的

# Scheduled Tasks — Claudeに繰り返し業務を任せる予約システム

## Scheduled Tasksとは？

毎朝Slackのメッセージを要約してほしい、毎週金曜日にプロジェクトレポートをまとめてほしい — 同じ依頼を繰り返しているなら、**一度設定して自動で動かせる**ようになりました。

Scheduled TasksはClaude Desktopの**Cowork**機能で使う予約実行システムです。作業の指示を一度書いておけば、設定したサイクルに合わせてClaudeが自動で実行し、結果を残してくれます。

> まとめ: Scheduled Tasks = **Claudeに設定するcronジョブ**

## ノーコードツールとの比較

| ノーコードツール | Scheduled Tasksの対応 |
|---|---|
| **ZapierのSchedule Trigger** | 時間ベースの自動実行（毎時、毎日、毎週） |
| **MakeのScenario予約** | 定期的にシナリオを実行 |
| **Google Apps Scriptの時間ベーストリガー** | 決まった時間にスクリプトを実行 |
| **n8nのCron Node** | cron式ベースの繰り返し実行 |

主な違い:
- ノーコードツールは**事前に作ったワークフロー**を繰り返し実行
- Scheduled Tasksは**自然言語の指示**を繰り返し実行 — ワークフローの設計なしで「こうしてほしい」と書くだけ

## どこで実行されるか

```
┌─ Claude Desktop（Cowork）─────────────────────┐
│                                                │
│  Scheduled Task: "毎日午前9時"                 │
│       │                                        │
│       ▼                                        │
│  Claudeが独立したCoworkセッションで実行          │
│       │                                        │
│       ├── MCPサーバーにアクセス可能（Slack、Driveなど）│
│       ├── プラグイン/スキルを使用可能            │
│       └── 結果をセッションに保存                │
│                                                │
│  ⚠️ 条件: コンピューターが起動中 + アプリが開いている │
└────────────────────────────────────────────────┘
```

**重要な制約**: Scheduled Tasksは**自分のコンピューターで実行**されます。コンピューターがスリープ状態またはClaude Desktopアプリが閉じていると実行がスキップされ、再び起動したときに自動で実行されます。

この点でZapier/Makeと根本的に異なります:
- Zapier/Make: クラウドで実行 → 自分のコンピューターの状態に無関係
- Scheduled Tasks: ローカルで実行 → アプリが起動している必要がある

## 実践シナリオ

### 1. 毎朝のブリーフィング
```
毎日午前9時
→ Slackの主要チャンネルのメッセージを要約
→ 今日のカレンダー予定を整理
→ 昨日未処理のイシューをリマインド
```

### 2. 週次レポート
```
毎週金曜日午後5時
→ Google Driveから今週のドキュメント変更点を収集
→ スプレッドシートデータをレポート形式に整理
→ 結果を指定フォルダに保存
```

### 3. 競合他社/業界動向の追跡
```
毎日
→ 特定のキーワードでニュースを検索
→ 主な発見事項を要約
→ トレンドの変化があればハイライト
```

### 4. ファイルの整理
```
毎週月曜日
→ ダウンロードフォルダのファイルを種類別に分類
→ 30日以上経過した一時ファイルの整理リストを作成
```

## 作成方法2種類

### 方法1: `/schedule` コマンド（対話形式）

Coworkで `/schedule` を入力すると、Claudeが会話で案内してくれます。

```
ユーザー: /schedule
Claude: どの作業を予約しますか？
ユーザー: 毎朝Slackを要約して
Claude: 実行頻度を選択してください → [毎時間] [毎日] [毎週] [平日のみ]
...
→ "Schedule"ボタンのクリックで確定
```

### 方法2: Scheduled Tasksページ（直接入力）

左サイドバーの **「Scheduled」** をクリック → **「+ New task」** をクリック

| フィールド | 説明 |
|---|---|
| Task name | 作業名（例: "Daily Slack Briefing"） |
| Description | 短い説明 |
| Prompt | 詳細な指示文（スキルのように自然言語で記述） |
| Frequency | 毎時間 / 毎日 / 毎週 / 平日のみ / 手動 |
| Model | 使用するモデルを選択（任意） |
| Working folder | 作業ディレクトリ（任意） |

## 管理機能

Scheduled Tasksページでできること:

- 実行履歴の確認（成功/スキップ/失敗）
- 指示文や周期の変更
- 一時停止 / 再開
- 手動でその場で実行
- 削除

## 使用条件

| 項目 | 条件 |
|---|---|
| プラン | Pro、Max、Team、Enterprise |
| プラットフォーム | Claude Desktopのみ（Web不可） |
| 実行条件 | コンピューターが起動中 + Claude Desktopが実行中 |

## まとめ

- Scheduled Tasks = **自然言語の指示を予約実行**するcronジョブ
- Zapier/Makeのように**ワークフローを設計する必要なく**、言葉で指示するだけ
- ただし、**ローカル実行**のためコンピューターが起動していなければならないという制約がある
- MCPサーバー、プラグイン、スキルと組み合わせて使用可能 → 外部サービス連携の自動化に活用
- クラウドベースの24/7自動化が必要な場合は引き続きZapier/Makeが適切

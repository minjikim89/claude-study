# Claude Code の体系 — 記憶構造と MD ファイルのレベル

## Claude Code の仕組み

Claude Code に初めて触れると「AI チャットボット」のように感じるかもしれませんが、実際には**ファイルシステム上で動作する AI エージェント**です。核心となるメカニズムはただひとつ：**コンテキストウィンドウ（200k トークン）にテキストを注入すること**。

Claude Code のすべての設定 — ルール、記憶、スキル — は最終的にマークダウンファイルであり、それらが適切なタイミングでコンテキストウィンドウに読み込まれる構造になっています。

## Chat Interface vs CLI — 根本的な違い

claude.ai（Web/アプリ）での会話体験を持って Claude Code に入門すると、似ているように見えても動作方式が根本的に異なることをまず理解する必要があります。

### Chat Interface（claude.ai Web/アプリ）

- **ブラウザやアプリ**のチャット画面にテキストを入力
- 会話内容は **Anthropic のクラウドサーバー**に保存
- Web で始めた会話をモバイルアプリで続けられる（同じアカウントであれば同期）
- AI ができること：テキスト生成、分析、要約など**会話内での作業**
- 自分のコンピューターのファイルにはアクセス不可

### CLI（Claude Code）

- **ターミナル**から `claude` コマンドで実行
- 会話内容は**自分のコンピューター（ローカル）**にのみ存在
- セッションを終了するとコンテキストは消滅（永続化は CLAUDE.md・Memory で別途対応）
- AI ができること：ファイル読み書き、ターミナルコマンド実行、Git 操作など**コンピューター全体を操作**
- 自分のプロジェクトフォルダ内で直接作業

### 比較一覧

| 項目 | Chat（claude.ai） | CLI（Claude Code） |
|------|-------------------|-------------------|
| 実行環境 | ブラウザ / モバイルアプリ | ターミナル（macOS、Linux） |
| 会話の保存 | クラウド（アカウント同期） | ローカル（セッション単位、同期なし） |
| ファイルアクセス | 不可（アップロードしたファイルのみ） | 全ファイルシステムにアクセス可能 |
| コード実行 | 不可 | Bash、Python 等を直接実行 |
| 外部ツール連携 | 限定的 | MCP で Slack、Notion、DB 等を接続 |
| 例え | LINEで秘書に質問 | 秘書が自分のオフィスに来て直接作業 |

**ポイント**：同じ Claude モデル（Opus、Sonnet など）を使いますが、**アクセスできる範囲**がまったく異なります。Chat は「会話の中」に閉じており、CLI は「コンピューター全体」を作業空間として使います。

## Chat と CLI は連携しない

Chat Interface での会話と CLI での会話は**完全に別々**です。

```
claude.ai（Web/アプリ）  ←→  同期あり  （同じアカウントなら会話を継続）
        ↕
       連携なし  ← 会話履歴・設定・コンテキストはすべて共有不可
        ↕
Claude Code（CLI）  →  ローカルセッション単位、終了時にコンテキスト消滅
```

| 項目 | Chat → CLI | CLI → Chat |
|------|-----------|-----------|
| 会話の引き継ぎ | 不可 | 不可 |
| 設定の共有 | 不可（Projects ≠ CLAUDE.md） | 不可 |
| ファイル共有 | 不可 | 不可 |

### なぜ分離されているのか？

Claude Code は**ローカルファイルシステム上で動作するエージェント**であるためです。会話自体がファイルの読み書きやターミナル実行などローカル環境と密接に結びついているため、クラウドベースの claude.ai と会話を共有することは構造的に困難です。

### ノーコードツールとの比較

| ノーコードツール | 同期方式 | Claude エコシステム |
|---|---|---|
| Notion Web ↔ Notion アプリ | 完全同期 | claude.ai Web ↔ アプリ：同期あり |
| Notion ↔ Zapier | API で接続（別製品） | claude.ai ↔ Claude Code：連携なし |
| Cursor ↔ ChatGPT | 完全別製品 | これと同様の関係 |

**まとめ**：claude.ai と Claude Code を Notion Web/アプリのように「同じツールの別接点」と考えてはいけません。むしろ **Notion と Zapier** のように「別製品」に近いです。背後で同じ AI モデルを使うだけで、インターフェース・ストレージ・機能はすべて異なります。

## コンテキストウィンドウの構造

```
┌─────────────────────────────────────────────────────────┐
│                    コンテキストウィンドウ（200k トークン）  │
│                                                          │
│  ┌──────────────┐                                        │
│  │ CLAUDE.md    │ ← 常に自動ロード。セッション開始と同時に注入│
│  │（グローバルルール）│                                   │
│  └──────────────┘                                        │
│                                                          │
│  ┌──────────────┐                                        │
│  │ Auto Memory  │ ← 常に自動ロード。MEMORY.md の内容を注入 │
│  │（学習記録）   │                                        │
│  └──────────────┘                                        │
│                                                          │
│  ┌──────────────┐                                        │
│  │ スキル A     │ ← "/コマンド" 入力時のみロード           │
│  │（レシピ）     │                                        │
│  └──────────────┘                                        │
│                                                          │
│  ┌──────────────┐                                        │
│  │ 会話内容     │ ← ユーザーと Claude のやり取りメッセージ  │
│  └──────────────┘                                        │
└─────────────────────────────────────────────────────────┘
```

200k トークンは無限ではありません。そのため**常に必要なものは自動ロード**、**たまに必要なものはオンデマンドロード**、**一時的なものは会話にのみ存在**するよう設計されています。

## CLAUDE.md — プロジェクトのルールブック

### 3 つのレベルシステム

CLAUDE.md は適用範囲によって 3 つのレベルで存在します。

```
~/.claude/CLAUDE.md           ← グローバル：すべてのプロジェクトに適用（個人の好み）
./CLAUDE.md                   ← プロジェクト：このプロジェクトに適用（チーム共有）
./CLAUDE.local.md             ← ローカル：このプロジェクト + 自分のみ（git 無視）
```

3 つのファイルがすべて存在する場合、**すべてを合わせて**システムプロンプトに注入されます。

### 各レベルの用途と例え

| レベル | ファイル位置 | 適用範囲 | 例え | 使用例 |
|---|---|---|---|---|
| **グローバル** | `~/.claude/CLAUDE.md` | 自分のすべてのプロジェクト | 個人の業務指針 | 「日本語で回答する」「テーブル形式を優先」 |
| **プロジェクト** | `./CLAUDE.md` | このプロジェクト全体 | チームコンベンション文書 | 「テストを先に実行」「コミットメッセージ形式」 |
| **ローカル** | `./CLAUDE.local.md` | このプロジェクト + 自分のみ | 自分のデスクのメモ | 「デバッグモードを常に有効」「特定パスを無視」 |

### ノーコードツールとの比較

| ノーコードツール | Claude Code |
|---|---|
| Notion ワークスペース設定 | グローバル CLAUDE.md |
| プロジェクトごとの設定ページ | プロジェクト CLAUDE.md |
| 個人ビュー / フィルター | CLAUDE.local.md |

### 作成のヒント

- **短く保つ**：毎セッション読み込まれるためトークンを節約する
- **命令形で書く**：「日本語で応答すること」「テストを先に実行すること」
- **構造化する**：見出しとリストで可読性を確保
- **優先度を明示**：競合する可能性があるルールは優先度を記載しておく

## セッション開始時に起こること

Claude Code を実行すると、以下の順序でコンテキストが構成されます：

```
新規セッション開始
  │
  ▼
┌─ 自動ロード（毎回）──────────────────────────────┐
│                                                  │
│  1. ~/.claude/CLAUDE.md          （グローバルルール） │
│  2. ./CLAUDE.md                  （プロジェクトルール）│
│  3. ./CLAUDE.local.md            （ローカルルール）   │
│  4. Auto Memory（MEMORY.md）     （学習内容）        │
│                                                  │
│  → すべて合わせてシステムプロンプトに注入          │
└──────────────────────────────────────────────────┘
  │
  ▼
┌─ 待機（必要なときのみ）────────────────────────┐
│                                                  │
│  5. スキル（SKILL.md 群）                         │
│     → リストのみ認識、本文は呼び出し時にロード    │
│                                                  │
└──────────────────────────────────────────────────┘
  │
  ▼
  会話開始
```

**ノーコードツールの例え**：アプリを開くと自動で設定が読み込まれるのと同じです。Notion を開くとワークスペース設定 + 個人設定が自動適用されるように、Claude Code もセッション開始時にすべての MD ファイルを読み込みます。

## 保存場所の全体マップ

```
~/.claude/
├── CLAUDE.md                          ← グローバルルール（すべてのプロジェクト）
├── settings.json                      ← Claude Code の設定
└── projects/
    └── <プロジェクト名>/
        └── memory/
            └── MEMORY.md              ← Auto Memory（プロジェクト別自動学習）

プロジェクトルート/
├── CLAUDE.md                          ← プロジェクトルール（チーム共有、git 管理）
├── CLAUDE.local.md                    ← ローカルルール（自分のみ、git 無視）
└── .claude/
    ├── skills/
    │   └── my-skill/
    │       └── SKILL.md               ← スキル（呼び出し時のみロード）
    └── agents/
        └── my-agent.md                ← カスタム Subagent 定義
```

## まとめ

- **Chat（claude.ai）と CLI（Claude Code）は別製品**：会話履歴・設定・ファイルはすべて共有されない。同じ AI モデルを使うが、インターフェースと機能はまったく異なる
- Claude Code のすべての設定は**マークダウンファイル → コンテキスト注入**のメカニズムで動作する
- CLAUDE.md は 3 つのレベル（グローバル / プロジェクト / ローカル）で適用範囲を制御する
- 常にロードされるもの（CLAUDE.md、Memory）と呼び出し時にロードされるもの（スキル）を区別してトークンを効率的に使用する
- セッション開始時は自動ロード → 待機 → 会話の順でコンテキストが構成される

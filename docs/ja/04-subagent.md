# Subagent — 独立した空間で働く部下

## Subagent とは？

Claude Code で会話している Claude は**メインエージェント 1 つ**です。しかし大きな作業をメインの会話で直接処理すると、コンテキストウィンドウ（200k トークン）があっという間に埋まってしまいます。

Subagent はこの問題を解決します：**独立した空間で作業し、結果だけをメインに報告**する構造です。

## ノーコードツールでの例え

| ノーコードツールの概念 | Claude Code Subagent |
|---|---|
| Zapier のサブ Zap（サブワークフロー） | メインの流れから分岐して別途作業を行い結果を返す |
| Make のサブシナリオ | 複雑なロジックを分離して処理 |
| Notion のリンクデータベース | 元データは別にあり、必要な情報だけ取得して表示 |

## なぜ必要か？

実際のシナリオで説明すると：

**Subagent なし** — 200 ページの PDF を分析してほしいと頼むと：
```
メインコンテキスト：[既存の会話] + [PDF 200 ページ全体] → トークン爆発
他の作業をしようとしてもコンテキストがすでに満杯
```

**Subagent 使用時** — 同じリクエスト：
```
メインコンテキスト：[既存の会話] + [要約 3 行のみ受信] → コンテキスト清潔
他の作業を継続可能
```

## 動作原理：Blank Slate

Subagent の核心的な特徴は **Blank Slate（白紙状態）** でのスタートです。

```
┌─ メイン Claude ──────────────────────────┐
│  会話コンテキスト：[A, B, C, D, E...]     │
│                                          │
│  "PDF を分析して" ──┐                     │
│                     ▼                    │
│  ┌─ Subagent ──────────────┐             │
│  │ コンテキスト：[blank slate]│ ← 空の状態から開始
│  │ 作業：PDF 分析            │             │
│  │ （200 ページを読む...）   │             │
│  │ 結果：要約 3 行 ──────────┼──▶ 転送    │
│  └─────────────────────────┘             │
│                                          │
│  メインは要約 3 行だけ受け取る → コンテキスト節約│
└──────────────────────────────────────────┘
```

**Blank Slate が意味すること：**
- メインの会話履歴を引き継がない
- Subagent はこれまでどんな会話をしてきたか知らない
- ただ**指示された作業に関する情報だけ**を持ってスタートする
- これが利点：不要なコンテキストなしに作業に集中できる

## Subagent の特性

| 特性 | 説明 |
|---|---|
| **Blank Slate** | メインの会話履歴を引き継がず、空の状態からスタート |
| **独立コンテキスト** | Subagent がファイルを 200 個読んでもメインのコンテキストはきれい |
| **結果のみ返却** | 作業過程は捨てられ、まとめられた結果だけをメインに報告 |
| **1:1 関係** | メインにのみ報告し、他の Subagent と直接通信不可 |
| **一回限り** | 作業完了後に消える（永続して存在しない） |

## 内蔵 Subagent の種類

Claude Code は用途別に最適化された内蔵 Subagent を提供します。

| Subagent | 使用可能なツール | モデル | 用途 | 例え |
|----------|----------------|------|------|------|
| **Explore** | 読み取り専用（Read、Glob、Grep） | Haiku（高速・低コスト） | ファイル探索、コードベース分析 | 図書館の司書 |
| **Plan** | 読み取り専用 | Sonnet/Opus | 実装計画の策定、アーキテクチャ設計 | 設計士 |
| **General-purpose** | すべてのツール | Sonnet/Opus | 複雑なマルチステップ作業 | 万能秘書 |
| **Bash** | Bash のみ | Sonnet/Opus | ターミナルコマンド実行 | システム管理者 |

### 各 Subagent の使いどき

**Explore** — 「このプロジェクトの構造はどうなってる？」
- コードベースを素早く探索
- 低コストな Haiku モデルを使用
- ファイルを修正しないため安全

**Plan** — 「この機能はどう実装すればいい？」
- コードを読んで計画だけを立てる
- 実際のコード作成はしない

**General-purpose** — 「この PDF を分析して要約して」
- すべてのツールを使えるので複雑な作業が可能
- ファイルの読み書き、ターミナル実行もすべて可能

**Bash** — 「テスト全体を実行して結果を教えて」
- ターミナルコマンドだけを実行
- 長いビルド/テスト過程の結果だけを報告

## カスタム Subagent の作成

内蔵 Subagent では不十分な場合、自分で作ることができます。

### ファイルの場所
```
プロジェクト/
└── .claude/
    └── agents/
        └── my-reviewer.md
```

### 作成例

```markdown
---
name: code-reviewer
description: コードの品質とセキュリティをレビューします
tools: Read, Grep, Glob
model: sonnet
---

あなたはシニアコードレビュアーです。

## レビュー項目
1. セキュリティ脆弱性（SQL インジェクション、XSS など）
2. パフォーマンスの問題（不要なループ、メモリリークなど）
3. コードの可読性
4. エラーハンドリングの漏れ

## 出力形式
| ファイル | 行 | 深刻度 | 内容 |
|------|------|--------|------|
```

### カスタム Subagent のメリット

- **ツールの制限**：`tools` フィールドで使用可能なツールを制限 → 安全な自動化が可能
  - 例：読み取り専用 DB 照会エージェント（Read、Grep のみ許可）
- **モデル選択**：シンプルな作業には低コストな Haiku、複雑な作業には Opus
- **役割の固定**：常に一貫した基準で作業を実行

## Subagent が自動的に使われる場合

ユーザーが明示的に Subagent を呼び出さなくても、Claude Code が判断して自動的に Subagent を起動する場合があります：

- コードベースの探索が必要なとき → **Explore** Subagent を自動生成
- 複雑な実装計画を立てるとき → **Plan** Subagent を自動生成
- 並列処理が効率的な作業のとき → 複数の Subagent を同時生成

## まとめ

- Subagent = **独立した空間で作業し、結果だけを報告**する下位エージェント
- Blank Slate でスタート → メインコンテキストの汚染を防止
- 4 種類の内蔵 Subagent（Explore、Plan、General-purpose、Bash）＋ カスタム可能
- カスタム Subagent は `.claude/agents/` にマークダウンファイルで定義
- メインと 1:1 関係 — 他の Subagent と直接通信不可（これが Agent Teams との違い）
